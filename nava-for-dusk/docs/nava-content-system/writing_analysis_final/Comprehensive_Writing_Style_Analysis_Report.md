## Part 4: Methodological Notes

### Analysis Techniques Used

1. **Readability Metrics**
   - Flesch Reading Ease: Measures text difficulty on 0-100 scale
   - Formula: 206.835 - 1.015(total words/total sentences) - 84.6(total syllables/total words)

2. **Vocabulary Analysis**
   - Type-Token Ratio (TTR): Unique words / Total words
   - Root TTR: Stemmed unique words / Total words
   - Hapax Ratio: Words appearing once / Total words

3. **Function Word Frequency**
   - Analysis of high-frequency English function words (the, of, and, to, a)
   - Rates calculated per 100 words for normalization

4. **Sentence Structure Analysis**
   - Average sentence length
   - Sentence length variance (standard deviation)
   - Punctuation patterns (commas, colons, semicolons, questions, exclamations)

5. **Engagement Markers**
   - Personal pronoun ratio (I, we, you)
   - Question frequency
   - Exclamation frequency

### Data Source

- **Input:** 76 markdown articles from `/home/tiny4lurv/.openclaw/workspace/rag_abroadworks/clean_md/`
- **Output:** Full analysis stored in `full_corpus_analysis.json`
- **Processing:** Python-based text analysis with NLTK for tokenization and syllable counting

### Limitations

1. Author self-identification in frontmatter — may include editorial changes
2. Sample size varies significantly (4-25 articles per author)
3. Some metrics sensitive to article topic (technical content naturally more complex)
4. Analysis treats each article equally regardless of publication date or performance
