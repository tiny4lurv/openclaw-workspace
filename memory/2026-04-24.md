## Persistent Issue — Agent Still Mentions Internal Resources
Tiny flagged at end of session: agent said *"If you want a full answer on that, I'd need to check the nava-notebook or get clarity from the source material."* This happened even after the INIT_MESSAGE rewrite. The rule is in the INIT_MESSAGE but the agent is ignoring it.

**Likely cause**: The agent is running in an OpenClaw session that has its own system context (SOUL.md, AGENTS.md, MEMORY.md, IDENTITY.md) which may include references to the notebooks. The relay's INIT_MESSAGE is sent as the first message in the session, but the agent's base instructions are loaded before that.

**Root lesson (from MEMORY.md "What Memory Instead of Reading Actually Means")**: The agent treats file reads as validation of what it already thinks, not actual investigation. Same pattern here — it has internal knowledge of the notebooks and retrieves it "from memory" rather than reading the INIT_MESSAGE instruction literally.

**This is the same core failure Dawn gets criticized for** — agreeing to a rule in one breath, breaking it in the next because "memory" told it otherwise.
