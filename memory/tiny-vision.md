_Not yet implemented. For reference and early thinking._

## Interview Mode

**What it does:** Dusk enters an structured interview with anyone knowledgeable about a topic. She takes notes, asks follow-ups, and forwards the findings to Tiny for review before anything goes live system-wide.

**Why it matters:** It's a controlled knowledge transfer engine — subject matter experts teach Dusk things through conversation, Tiny reviews and approves, then it propagates. Built-in human review gate prevents bad knowledge from spreading.

**Safety:** Review step is mandatory. Tiny reviews all interview outputs before deployment. This prevents harmful or incorrect information from entering the system.

**Current scope:** Reclaimix process. **Future scope:** Anything Nava recruits for — including non-clinical roles.

**Status:** Not yet built. I need to start thinking about how to design this.

## Automation / Tool Request Mode

**What it does:** Team members describe workflows they want automated. Dusk and Dawn help design the tool. Tiny builds it.

**Current proof of concept:** The Elfsight replacement for the Nava job board. Download → filter → compare → update → done, without manual spreadsheet checking.

**What this means in practice:** Team tells Dusk what shortcuts they wish they had. Conversations with Dusk become Tiny's discovery and programming brief.

**Status:** Not yet live for team. Design framework TBD.

## Admin Mode (Authentication)

**What it does:** Passphrase + laptop fingerprint = admin access. Hardware-bound so credentials can't be used on another machine.

**What you get in admin mode:**
- Interview mode
- Automation / tool request mode
- Other functions still in development

**Who gets access:** The test group eventually becomes admins.

**Status:** Not yet built. Authentication design Tiny is working on.

## Pop-Up Dusk (Browser Extension)

**What it does:** A persistent pop-up button that follows you to any page. Reads the current tab, summarises it, lets you ask questions. Personal diary of things you find and need for work.

**Analogy:** Grammarly pop-up model — one button, works everywhere, doesn't require switching tools.

**Key features:**
- Read and summarise any page you're on
- Ask page-specific questions
- Keep a personal diary / memory of things you find
- Eventually: works on any page, not just Nava

**Near-term scope:** Nava website only (controlled content, predictable structure).

**Hard problem:** Any page means handling SPAs, dynamic JavaScript content, paywalled articles, etc. Not all pages are clean HTML.

**Status:** Not yet built. Widget is the current constrained version.
